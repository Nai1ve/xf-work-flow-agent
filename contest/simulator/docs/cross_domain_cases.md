# 综合题（Cross-domain）Case 列表

本文汇总 `beta_zh_*` 综合题，说明每道题涉及的工具域、核心考点和当前设计原则。

## 设计目标

- 考察 Agent 在多个工具域之间切换和保持上下文的能力
- 避免只靠单一模板完成任务
- 允许“部分任务成功 + 部分任务阻塞”的真实办公场景
- 保持每道题仍然可评分、可复现、可解释

## 当前范围

当前已覆盖两段综合题：

- 训练集：`beta_zh_0001 ~ beta_zh_0038`
- 验证集：`beta_zh_0201 ~ beta_zh_0230`

主要涉及：

- `meetingroom`
- `workflow`（请假、费用类物资申请）
- `user`
- `oa`

## 能力覆盖

### 会议室侧

- 基础预订
- A1/A2 fallback
- 最近会议室优先
- 已有会议取消后重订
- 会议延长
- 添加参会人
- 模糊取消唯一定位
- 无权限会议室阻塞
- 延长失败但保持原会议不动

### workflow 侧

- 请假草稿保存
- 请假直接提交
- 审批人歧义阻塞
- 费用类草稿保存
- 费用类直接提交
- 项目歧义 / 子类歧义下的正确选择或阻塞
- 多明细费用提交
- draft / submit / blocked 三种状态并存

### 辅助工具

- `user.get_workspace`
- `user.get_info`
- `oa.todo.list`
- `oa.done.list`

## 评分设计原则

- 每道题通常拆成两个主子任务，分别写入 `success_check.must_satisfy`
- 阻塞型子任务会明确要求：
  - 不得错误写入
  - `final_answer` 返回 `blocked`
- 会议室子任务继续遵守：
  - 唯一合法候选
  - 不得保留额外新增活跃会议
- workflow 子任务继续遵守：
  - 草稿 / 提交状态与题意一致
  - 关键字段、项目、类别、小类、明细值正确

- `beta_zh_0001 ~ 0005` 适合作为综合题公开样例
- `beta_zh_0006 ~ 0010` 适合作为中等难度综合题
- `beta_zh_0011 ~ 0015` 更适合作为隐藏测试或进阶综合题
- `beta_zh_0016 ~ 0020` 适合作为“部分阻塞”公开样例
- `beta_zh_0021 ~ 0025` 适合作为“一个任务成功、一个任务阻塞”的验证题
- `beta_zh_0026 ~ 0030` 更适合作为隐藏测试或高难综合题
- `beta_zh_0031 ~ 0038` 适合作为训练集补强样板，重点覆盖部分阻塞、会议变更和费用/请假并行处理

## 综合题验证集（beta_zh_0201 ~ 0230）

这 30 道验证集刻意与训练集错位，不再重复“订会议室 + 顺手存草稿”这类模板，而是更强调：

- 一个子任务成功，另一个子任务必须明确 `blocked`
- 会议室变更链路与 workflow 选择/阻塞链路同时出现
- `user`、`oa`、`meetingroom`、`workflow` 多工具切换
- 更少依赖固定 query 模板，更考验 Agent 的泛化能力

### 第一组：部分阻塞型（`beta_zh_0201 ~ 0210`）

- 一个子任务应成功完成
- 另一个子任务必须因为歧义、权限或信息不足而阻塞
- 核心考察：
  - 不能因为一个任务 blocked 就放弃另一个
  - 也不能为了“尽量都完成”去乱保存、乱预订

### 第二组：多步变更型（`beta_zh_0211 ~ 0220`）

- 至少一个子任务包含：
  - `cancel`
  - `extend`
  - `rebook`
  - `participant.add`
  - `oa.todo.list / oa.done.list`
- 核心考察：
  - 是否能保持原状态一致性
  - 是否会在多步动作里引入额外错误写操作

### 第三组：泛化选择型（`beta_zh_0221 ~ 0230`）

- 更强调 workspace 近场优先、会议室 fallback、费用项目歧义、费用子类歧义、请假审批人歧义的组合
- 核心考察：
  - 是否真的理解工具和 schema
  - 是否能在多个“看起来都合法”的路径里选语义最正确的一条
  - 是否在无法唯一判断时明确阻塞

### 验证集建议用法

- `beta_zh_0201 ~ 0210` 适合作为部分阻塞验证题
- `beta_zh_0211 ~ 0220` 适合作为高难变更链路验证题
- `beta_zh_0221 ~ 0230` 更适合作为隐藏测试或泛化测试题
